SafeWalk

Modeling & Evaluation

Model Exploration

  • YOLOv5s: Selected for real-time speed and mobile compatibility
  • YOLOv11s: Higher accuracy but increased inference time
  • Faster-RCNN: Explored but rejected due to long training time and poor mobile support

Final Model Performance

  • Precision: 0.63
  • Recall: 0.51
  • mAP@0.5: 0.51
  • Latency: Optimized for on-device detection

Visual Language Model (VLM)

  • Model: Qwen2 selected over Paligemma for richer descriptions
  • Qwen2 vs Paligemma: +1.7 points in spatial orientation and environmental awareness
  • Qwen2 averaged 65–70 words per description vs 11–12 with Paligemma

Model Evaluation & Metrics

  • Tracked metrics: Precision, Recall, mAP@0.5, mAP@0.5:0.95
  • Reviewed class-level confusion matrices to address edge-case failure
  • Latency testing on Android and iOS for alert responsiveness

Key Technical Challenges

  1. Imbalanced training data across 28 custom object classes
  2. Meeting latency requirements for mobile while preserving accuracy
  3. Integrating on-device and cloud-based components in one UX

Future Roadmap

  • Deploy to smart glasses or smart cane systems
  • Refine voice alert logic through user testing
  • Improve multilingual support for international users

SafeWalk

Technical Highlights

Overview

SafeWalk leverages computer vision and large language models to support visually impaired individuals in real-time navigation. The app combines a custom-trained object detection model and a visual language model to deliver obstacle detection, voice alerts, and rich scene descriptions.

Data & Modeling Approach

  • Training dataset: ~15GB, 91,000+ files, 28 object classes (e.g., potholes, scooters, bollards, stairs, etc.)
  • Preprocessing with Roboflow, manual validation to avoid poor labeling
  • Addressed class imbalance using data augmentation and undersampling
  • Replay-based learning used to mitigate catastrophic forgetting during transfer learning

Model Selection

We evaluated Faster-RCNN and two YOLO variants (v5s and v11s). YOLOv5s was selected due to:

  • Lower training time and cost (20 min per epoch vs. 3.5 hrs for Faster-RCNN)
  • Higher deployment efficiency on mobile devices (supports TFLite, CoreML, ONNX)
  • Balanced performance: Precision 0.63, Recall 0.51, mAP@0.5: 0.51

Visual Language Model (VLM)

  • Compared Paligemma and Qwen2; Qwen2 outperformed with longer, more detailed scene descriptions
  • Key metric improvements: +1.7 in Spatial Orientation, +1.7 in Environmental Awareness
  • Descriptions averaged 65–70 words (vs. 11–12 in Paligemma)

Evaluation & Metrics

  • Metrics tracked: Precision, Recall, mAP@0.5, mAP@0.5:0.95, latency
  • Confusion matrix revealed class-level accuracy gaps due to data imbalance
  • Latency optimized for real-time alerts on smartphones

Key Technical Challenges

  1. Managing 28 object classes with imbalanced training data
  2. Ensuring real-time latency with limited compute power on mobile devices
  3. Maintaining model accuracy while extending class variety

Future Roadmap

  • Integrate with smart glasses and canes for hands-free experience
  • Refine alert rule engine with user feedback
  • Expand multi-language and accessibility support

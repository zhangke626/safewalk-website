SafeWalk

Data & Preprocessing

Our data pipeline powers SafeWalk's ability to understand and react to complex urban environments in real-time.

Dataset Summary

  • Volume: ~15GB total, over 91,000 images
  • Classes: 28 custom object classes including pothole, scooter, stairs, tree, fire hydrant, etc.
  • Sources: Public datasets, web scraping, real-world photos

Preprocessing Workflow

  • Labeled with Roboflow and manually verified
  • Data augmentation for minority classes (rotation, flipping, brightness adjustment)
  • Undersampling of dominant classes (e.g., person) to balance dataset
  • Split: 80% train, 10% validation, 10% test

Handling Class Imbalance

  • Used image-weights in loss function to emphasize underperforming classes
  • Custom learning rate scheduling to prevent overfitting on dominant classes

Replay-Based Learning

To prevent catastrophic forgetting when introducing new object classes, we implemented replay-based learning. This technique retains original class accuracy while adding new categories efficiently.

Data Challenges

  • Mislabeling and noise in public datasets
  • Minority classes like potholes were hard to collect and label
  • Needed to maintain quality while scaling dataset size

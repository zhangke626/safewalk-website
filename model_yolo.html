YOLOv5s: Obstacle Detection Model

We selected and fine-tuned YOLOv5s as our base for object detection due to its lightweight structure, fast inference speed, and strong performance. This model enables real-time obstacle detection on mobile devices.

Fine-Tuning Strategy

  • Changed optimizer from SGD to Adam
  • Lowered learning rate from 0.01 to 0.001
  • Increased training epochs to 100
  • Implemented label smoothing with value 0.1
  • Used class-specific thresholds (e.g., Car: 0.35, Bollard: 0.50)
  • Addressed class imbalance with undersampling and augmented oversampling
  • Incorporated class weights for rare classes

Smart Alert System

Our adaptive alert engine uses YOLO outputs to trigger spoken warnings based on real-time detection and object location. This system aims to enhance safety while minimizing alert fatigue.

  • Alerts only trigger in critical moments
  • Direction-specific cues: e.g., “Bicycle on your right” or “Obstacle in front”
  • Bounding box parameters (center width, top, height) are used for decision-making
  • Class-based logic tailored for objects of varying sizes like trees, people, or bollards

Class-specific bounding box logic enables contextual and directional alerts

Key Challenges & Learnings

Developing consistent alerting rules for object classes with drastically different shapes and sizes—such as wide cars vs. narrow bollards—required extensive testing and adjustments.

We used a custom test mode to evaluate how bounding boxes behaved across real-world conditions and tuned logic accordingly.

Bounding box parameters like center width and height helped us differentiate class-based behaviors
